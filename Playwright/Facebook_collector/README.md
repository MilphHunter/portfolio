Даний код парсить Фейсбук, використовуючи приватний, інкогніто веб-браузер Vision, до якого підключається з використанням api.

Підключаючись до папки (через його айді) з «профілями» фб користувачів, код поступово отримує інформацію про «користувача»,
після чого ініціалізує запуск браузера під конкретний «профіль».

Спочатку, отримавши через API проксі для країни профілю, ми перевіряємо їх 
працездатність, використовуючи обробник помилок, після чого пробуємо увійти в фб
використовуючи кукі.
При успішному проходженні перевірок Фейсбука, відбувається скрол сторінки вниз,
поступово збираючи рекламу через вбудовані розширення браузера, так само аналізуючи
їх кількість. Після закінчення певного часу або при мінімальній кількості реклами,
парсинг припиняється і береться новий профіль.

В кінці код відправляє статистику, дані і позначаємо профіль як відпрацьований,
щоб відправити його на «відопочинок» на конкретний час.

В папці json_api знаходятся 2 файла, які взаємодіють з FastAPI сервером.
"current_profiles.json" -- зберігає в собі профілі, які в данний момент оброблює код.
"settings.json" -- налаштування, які код отримав використовуючи FastAPI сервер.

Весь код працює через тредси, тому можна задати певну кількість профілів, які будуть оброблятися одночасно.
Код працює асинхронно, також застосовуючи async playwright.
