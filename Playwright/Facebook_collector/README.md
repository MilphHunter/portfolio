### Опис роботи

Даний код виконує парсинг Facebook, використовуючи приватний інкогніто-браузер **Vision**, до якого відбувається підключення через **API**.

Підключаючись до директорії з *профілями* Facebook-користувачів (за їхнім ID), код поступово отримує інформацію про кожен профіль та ініціалізує запуск браузера під конкретного користувача.

---

### Логіка роботи

1. Через API отримується проксі для країни, до якої належить профіль.  
2. Після цього виконується перевірка працездатності проксі з використанням обробника помилок.  
3. Далі здійснюється спроба входу у Facebook через збережені **cookies**.  
4. Якщо перевірки Facebook проходять успішно, запускається скрол сторінки вниз — браузер поступово збирає рекламу через вбудовані розширення, одночасно аналізуючи її кількість.  
5. Коли спливає заданий час або реклама майже відсутня, парсинг завершується, і код переходить до наступного профілю.

Після завершення обробки профілю код відправляє зібрані дані та статистику, а профіль позначається як опрацьований і відправляється на «відпочинок» на визначений період.

---

### Взаємодія з FastAPI

У директорії **`json_api`** знаходяться два файли:

- **`current_profiles.json`** — зберігає профілі, які зараз обробляються.  
- **`settings.json`** — налаштування, отримані через FastAPI сервер.

---

### Паралельна обробка

Увесь код працює через **threads**, що дозволяє задавати кількість профілів, які обробляються одночасно.  
Також використовується **асинхронний Playwright (async)** для підвищення продуктивності.
